How It Works

Two qualification states — no ambiguity

✅
100% Confirmed

No action needed

All criteria verified in Banner, FAFSA, and CCCApply data. Student is immediately enrolled or added to the priority queue.

Example: EOPS Tier 1 — income <$19k + homeless + low GPA

📋
Conditionally Qualified

One item needed

Core criteria met. One supplemental document or form is required. Banner tells the student exactly what — no guessing.

Design Decision

Why rules-based, not AI

❌ Agentic RAG would give us…

  • Unpredictable — same student, different result each run
  • Hallucination risk on legal eligibility criteria
  • Cost per page load — every student visit charges tokens
  • No audit trail — can't explain why a student was told yes

✅ Rules engine gives us…

  • Deterministic — same profile, same result, every time
  • Free — zero API cost per student check
  • Instant — runs in milliseconds server-side
  • Fully auditable — every decision traced to a criterion

EOPS eligibility is defined by California Title 5 §56220. The criteria are fixed and binary. AI earns its place for open-ended questions — not legal checklists.

Implementation Path

How GWC plugs this in

Technical — 2 weeks

  • Read-only access to 6 Banner SIS fields via Ethos API
  • Read BOG waiver + income bracket from Financial Aid module
  • Replace mock profile in page.tsx with real auth session (SSO/LMS)
  • Deploy to GWC subdomain via Vercel — one click from GitHub

Organizational — 6–10 weeks

  • EOPS director policy: Banner-verified = sufficient for auto-enrollment
  • Financial Aid: one data-sharing agreement for BOG waiver field
  • IT: one FERPA security review (covers all future students)
  • Approve once — not 9,000 times

Extension path: start EOPS → add CalWORKs → add CARE → add NextUp, DSPS, Golden Promise. Same codebase, no rewrite.
